Voici une explication claire, structurée et illustrée à partir des documents que vous avez fournis, sur les différents types d’erreurs/biais d’un modèle, comment les estimer, et quelles approches permettent de les réduire en machine learning.

🔍 1. DÉFINITION DES DIFFÉRENTS TYPES D’ERREURS / BIAIS DANS UN MODÈLE

En apprentissage supervisé, l’erreur totale d’un modèle peut être décomposée en trois composantes principales :

➤ a) Erreur d’approximation (ou biais)

  • C’est la différence entre la vraie fonction cible f(x)f(x)f(x) et la fonction moyenne apprise par l’algorithme.

  • Elle est élevée quand le modèle est trop simple pour capturer la complexité de la tâche (ex. : régression linéaire sur une courbe non linéaire).

  • Source : hypothèses trop fortes ou espace de recherche trop restreint.

➤ b) Erreur d’estimation (ou variance)

  • Mesure la sensibilité de l’algorithme aux variations des données d’entraînement.

  • Élevée quand le modèle est trop flexible (ex. : arbres de décision profonds, k-NN avec k=1).

  • Source : petit jeu de données ou modèle trop complexe.

➤ c) Erreur irréductible (bruit) – Bayes error

  • Liée à la nature aléatoire de certaines données (y=f(x)+ϵy = f(x) + \epsilony=f(x)+ϵ) : même le meilleur modèle ne peut la réduire.

  • Elle est inhérente au problème, et ne dépend pas du modèle.

Formule de décomposition :

E[(y−f^(x))2]=Biais2+Variance+Bruit\mathbb{E}[(y - \hat{f}(x))^2] = \text{Biais}^2 + \text{Variance} + \text{Bruit}E[(y−f^​(x))2]=Biais2+Variance+Bruit

📏 2. COMMENT ESTIMER CES ERREURS EN PRATIQUE

➤ a) Erreur d’entraînement (EinE_{in}Ein​)

  • Calculée sur le même jeu de données utilisé pour l’apprentissage.

  • Toujours optimiste (le modèle a été optimisé pour ces données).

➤ b) Erreur de test (ou de généralisation) (EoutE_{out}Eout​)

  • Estimation de la performance sur des données jamais vues.

  • Peut être obtenue avec :

    • Hold-out : on divise le jeu de données en train/test

    • Validation croisée (cross-validation) : on divise en k sous-ensembles, on apprend sur k-1 et on teste sur le 1 restant

    • Bootstrap : tirage avec remise pour créer plusieurs jeux de train/test​

➤ c) Estimation du biais et de la variance

  • En répétant l’apprentissage sur plusieurs échantillons indépendants, on peut estimer :

    • Biais2=(E[f^(x)]−f(x))2\text{Biais}^2 = (\mathbb{E}[\hat{f}(x)] - f(x))^2Biais2=(E[f^​(x)]−f(x))2

    • Variance=E[(f^(x)−E[f^(x)])2]\text{Variance} = \mathbb{E}[(\hat{f}(x) - \mathbb{E}[\hat{f}(x)])^2]Variance=E[(f^​(x)−E[f^​(x)])2]

  • Cela nécessite un grand nombre de jeux de données simulés ou des techniques comme le bootstrap​

🧠 3. MÉTHODES THÉORIQUES ET ALGORITHMIQUES POUR RÉDUIRE L’ERREUR DE GÉNÉRALISATION

➤ a) Techniques pour réduire le biais

  • Utiliser des modèles plus complexes (régression polynomiale, réseaux de neurones)

  • Augmenter l’espace d’hypothèses

  • Booster les performances (ex : Adaboost dans les premières itérations)​

➤ b) Techniques pour réduire la variance

  • Bagging : combiner plusieurs modèles appris sur des sous-échantillons bootstrap (ex : forêts aléatoires)

  • Pruning (élagage des arbres)

  • Régularisation : pénaliser la complexité (L1/L2)

  • Augmenter la taille du jeu de données

  • Réduction de la dimension (ex : PCA)​

➤ c) Équilibre biais-variance

  • Le choix du modèle repose sur un compromis : plus flexible ⇒ moins de biais, mais plus de variance.

  • L’objectif est de minimiser l’erreur totale, pas uniquement l’un ou l’autre​

🧠 4. MÉTAPHORE INTUITIVE

Imaginez que vous essayez de tirer à l’arc sur une cible :

  • Haute variance : vos flèches sont dispersées dans toutes les directions.

  • Haut biais : toutes vos flèches sont regroupées… mais loin du centre.

  • Bon modèle : vos flèches sont regroupées et proches du centre.

🧪 5. RAISONNEMENT AVANCÉ ET EXPLOITATION

➤ a) Supposons qu’un SVM a zéro erreur d’entraînement avec k vecteurs de support

  • En leave-one-out cross-validation, l'erreur maximale est k/n

  • Cela montre que plus un modèle est "sparse", plus il peut bien généraliser même sans erreur d’entraînement​

➤ b) Que se passe-t-il si on augmente la capacité du modèle ?

  • La variance augmente (plus sensible aux données d’entraînement)

  • Le biais diminue (meilleure capacité à approximer des fonctions complexes)

  • Il faut donc réguler, par exemple avec une pénalité L2 dans la régression ridge​

🎓 6. QUESTIONS D’EXAMEN POSSIBLES

Q1. Quelle est la différence entre biais et variance ?
R : Le biais mesure l’incapacité à modéliser correctement la fonction cible. La variance mesure la sensibilité aux variations du jeu de données.

Q2. Pourquoi la validation croisée donne-t-elle une meilleure estimation de l’erreur hors échantillon que l’erreur d’entraînement ?
R : Parce qu’elle teste le modèle sur des données qu’il n’a pas vues, ce qui simule la généralisation.

Q3. Expliquez pourquoi bagging réduit la variance.
R : Parce qu’il moyenne les prédictions de plusieurs modèles appris sur des jeux de données différents, ce qui compense les fluctuations individuelles.

X_filtered = X_var_df.drop(columns=to_drop_corr)
